## 🎯 Solutions Testées

### ✅ Solutions Réussies

1. **Réorganisation du Router**
   - Priorité 1: Cases spécifiques (2-33)
   - Priorité 2: Cases par défaut (1, 8)
   - Résultat: Case 25 détecté correctement

2. **Code Markdown Generator Corrigé**
   - Extraction de `data.output` (priorité 1)
   - Gestion de 3 structures différentes
   - Résultat: Pas d'erreur "data ne contient pas un array"

3. **Format Markdown Simple**
   - Remplacement HTML par Markdown
   - Format: `**En-tête**\n\nContenu`
   - Résultat: Tables 2-5 affichées correctement

### ❌ Solutions Échouées

1. **Format HTML pour Tables 2-5**
   - Tentative: Utiliser `<table>` avec styles CSS
   - Échec: HTML non rendu par le composant Markdown
   - Leçon: Utiliser Markdown natif plutôt que HTML

2. **Prompt avec Exemples Détaillés**
   - Tentative: Montrer des exemples complets dans le prompt
   - Échec: LLM copie les exemples au lieu de générer du contenu
   - Leçon: Utiliser des instructions génériques sans exemples

3. **Prompt avec Placeholders**
   - Tentative: Montrer la structure avec `[Titre...]`
   - Échec: LLM retourne les placeholders littéralement
   - Leçon: Ne jamais montrer de placeholders au LLM

## 🎓 Leçons Apprises

### Router Front-End
- Toujours vérifier les cases spécifiques AVANT les cases par défaut
- Utiliser des conditions explicites plutôt que des else if imbriqués

### Workflow N8N
- Ne jamais mettre d'exemples hardcodés dans les prompts système
- Ne jamais montrer de placeholders au LLM (il les copie)
- Toujours débugger avec des logs pour voir ce qui est passé au LLM
- S'assurer que le message utilisateur est INCLUS dans le prompt

### Frontend ClaraVerse
- Préférer Markdown natif plutôt que HTML pour la compatibilité
- Tester l'affichage de toutes les tables, pas seulement la première
- Vérifier que le format de réponse est conforme au FORMAT 4

### Débogage
- Isoler chaque composant (router, webhook, prompt, LLM, réponse)
- Vérifier les logs à chaque étape
- Tester avec des messages simples d'abord
